 [💬 TG商务合作](https://t.me/cy_MariaBuild) · [📢 TG项目频道](https://t.me/Maria_Build)

> 本仓库仅用于项目功能展示与商务合作介绍，不提供核心源码、数据库、Session及一键安装包。

一套用于 Telegram 群组、频道及私聊消息关键词监控的独立部署机器人。

监听账号发现指定关键词后，系统会立即向管理员推送消息来源、发送者信息、消息内容及原消息链接，方便及时查看和处理目标信息。

## 主要功能

- 支持监听群组、频道及私聊消息
- 支持包含关键词和精准关键词
- 支持关键词单独添加及批量添加
- 支持通过 TXT 批量导入、导出关键词
- 支持监听账号已经加入的全部群组和频道
- 支持仅监听指定群组或频道
- 支持批量导入群组、频道用户名或邀请链接
- 支持扫码登录监听账号
- 支持手机号及验证码登录
- 支持 2FA 二步验证
- 支持 Telethon StringSession 登录
- 支持批量导入备用 Session
- 支持用户黑名单
- 支持群组及频道黑名单
- 支持自定义历史上下文条数
- 支持自定义重复推送间隔
- 支持忽略机器人消息和监听账号自己的消息
- 命中通知显示发送者、用户名、消息来源和时间
- 支持跳转查看原消息
- 支持快捷屏蔽用户或消息来源
- 支持关键词命中记录导出 Excel
- 历史数据自动定期清理
- 支持独立 VPS 部署
- 支持 systemd 开机自启
- 支持在线自检、日志查看和数据维护

## 关键词匹配方式

### 包含关键词

消息中只要包含指定关键词，即可触发通知。

例如设置关键词：

```text
机器人搭建
```

下面这条消息可以命中：

```text
请问有人可以做机器人搭建吗？
```

### 精准关键词

只有消息内容与设置的关键词完全一致时，才会触发通知。

例如精准关键词：

```text
机器人搭建
```

只有消息完整内容为“机器人搭建”时才会命中。

## 监听范围

可根据实际需求选择：

- 监听账号已经加入的全部群组和频道
- 仅监听指定目标列表中的群组和频道
- 单独开启或关闭私聊监听
- 单独开启或关闭群组监听
- 单独开启或关闭频道监听

监听账号只能读取自己已经加入，并且拥有正常访问权限的群组或频道消息。

## 监听账号登录

支持以下登录方式：

- Telegram 扫码登录
- 手机号及验证码登录
- 2FA 二步验证
- Telethon StringSession 登录
- TXT 批量导入备用 Session

当前独立部署版本一次启用一个主监听账号，其他 Session 可作为备用账号保存和切换。

## 命中通知

关键词命中后，管理员可以查看：

- 命中的关键词
- 发送者名称
- 用户 ID
- Telegram 用户名
- 用户类型
- 消息来源名称及用户名
- 消息发送时间
- 消息正文
- 原消息链接

通知下方支持发起私聊、查看原消息、屏蔽用户、屏蔽群组或频道以及删除通知。

部分私有群组、私有频道或私聊消息受 Telegram 权限限制，可能无法生成可公开访问的消息链接。

## Excel 数据导出

支持将关键词命中记录导出为 Excel 文件，内容包括：

- 发送时间
- 用户名称
- 用户 ID
- 用户名
- 用户类型
- 发送内容
- 消息来源
- 会话链接

导出文件成功发送后，本次已导出的旧记录会被清理，请及时下载并妥善保存。

## 部署说明

本项目采用单客户、独立 VPS 部署方式。

每位客户独立拥有：

- 管理机器人
- 监听账号
- 关键词数据库
- 监听目标
- 黑名单
- 消息记录
- Session 文件
- 系统运行服务

客户通常需要准备：

- Debian 12、Ubuntu 22.04 或 Ubuntu 24.04 VPS
- Telegram 管理机器人 Bot Token
- Telegram API ID
- Telegram API Hash
- 管理员 Telegram 数字 ID
- 用于监听的普通 Telegram 账号
